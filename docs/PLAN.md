# План решения задачи по фурье-фильтрации

## Цель
Найти частоты полезного сигнала в зашумлённом бинарном файле измерений, отфильтровать шум, восстановить полезный сигнал и построить требуемые графики.

## Входные данные
- Бинарный файл с `N = 100000` отсчётами.
- Интервал дискретизации: `0.1 ms = 0.0001 s`.
- Присутствует случайный шум.
- Амплитуды гармоник полезного сигнала как минимум в два раза больше амплитуд гармоник шума.

## Пошаговое решение
1. Считать бинарный файл и преобразовать сырые байты в числовой массив.
2. Проверить, что файл содержит ровно `100000` отсчётов.
3. Построить временную ось:
   - `t_k = k * 0.0001 s`
4. Вычесть среднее значение сигнала, чтобы убрать постоянную составляющую.
5. Применить БПФ для перехода из временной области в частотную.
6. Построить амплитудный спектр.
7. Оценить уровень шума с помощью медианы амплитуды спектра.
8. Найти локальные максимумы, амплитуда которых не меньше `2 * noise_floor`.
9. Считать эти пики частотами полезного сигнала.
10. Оставить только бины вблизи найденных пиков, а остальные подавить.
11. Выполнить обратное БПФ для восстановления отфильтрованного сигнала.
12. Сохранить:
    - список найденных частот,
    - отфильтрованный сигнал,
    - спектр,
    - графики.

## Ожидаемые выходные файлы
- `detected_frequencies.txt`
- `summary.json`
- `filtered_signal.csv`
- `spectrum.csv`
- `signal_time_domain.png`
- `filtered_signal.png`
- `spectrum.png`

## Примечания
- Проект поддерживает автоматическое определение `dtype` по размеру файла.
- Если реальный формат файла неоднозначен, укажи `--dtype` вручную.
- Порог поиска частот по умолчанию использует условие задачи: полезные пики должны быть как минимум в два раза сильнее шума.
